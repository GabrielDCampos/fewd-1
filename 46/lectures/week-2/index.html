Floats

When you want an image and text to appear side-by-side, you want to `float` the image (usually) to the `left` or `right`.

The balloon to the right is float: right.

Once an element is floated, it is no longer part of the "document flow," which can often have unintended consequences.

Clearfix

A "clearfix" is a hack used to manage layouts with floats, see this example. The modern way to do this is by adding the CSS property overflow: auto; to the parent.

Positioning

When you use anything other that static (the default value for position), you'll be able to use the properties top, right, bottom and left to adjust your elements position, but how it is positioned depends on what value you set position to:

  • position: static;

    Positions based on the content; cannot be adjusted and it's the default value for position for all elements (you probably don't need to know that, but can use it to sound cool at parties).

  • position: fixed;

    Positions with respect to the browser window (useful for sticky headers and stuff). Use with caution because it will probably cover text!

  • position: relative;

    Positions relative to where it normally is.

  • position: absolute;

    "Absolute children have relative parents!" An absolute child element will be positioned relative to it's parent (the one that has position: relative).

Classes & IDs

Classes and IDs allow us to identify and target styles on elements based on more than just their tag name. Let's say you have a particular button style you like. You'd like to be able to style <button> and <a> in the same way. One way to do this is to attach your styles to both elements: 

a,
button {
  border: 1px solid black;
  border-radius: 5px;
  /* etc... */
}

What's bad about this?

Creating a class allows apply styles to any element as we choose. Define a class with a .: 

.my-button {
  border: 1px solid black;
  border-radius: 5px;
  /* etc... */
}

And in your HTML, give elements that class with the class attribute. 

<a href="http://example.com" class="my-button">Click me<a>

Sharing styles and writing less code

An element can have as many classes modifying it as you'd like. This let's us share common styles and get more specific as we need to. 

.my-button {
  border: 1px solid black;
  border-radius: 5px;
  font-size: 16px;
  padding: .25em;
}
.my-smaller-button {
  font-size: 12px;
}

IDs

IDs work just like classes but are less flexible because an element can only have one ID. You also use a # instead of a . to select an element by ID.

Display: block & inline

Block elements are "100% wide" and do not let any other pieces of content share space on the same horizontal row.

Inline elements are as wide as their content and allow other elements to share space to the left and right.

div & span

When you can add whatever descripitors you want to an element via classes or IDs, the name of the element becomes less important. For example, you could have a paragraph that serves as the "introduction" to an article. If you had a <introduction> tag, you might use it instead. In cases like this, consider using a completely generic element like <span> and <div>.
