5. GitHub

What is "source control?"

Source control is a system of managing the code we write. We don't just write code and leave it forever. When you want to make an improvement or fix a bug, you have to edit code you've written. Using source control makes that process easier, especially when working on a team.

  • If you're a designer or writer, how do you version your files?
    • What's good about that system?
    • What's bad about that system?
    • What happens when you're dealing with huge files, like videos?
  • What happens when you want to work on multiple things at once?
  • What are some other flavors of source control?
  • What's a "repository?"
  • What's a "commit?"
  • What's a "diff?"
  • What's a "branch?"
  • What's the difference between git and GitHub?
  • What is git actually doing under the hood?

The anatomy of an element

At its most essential, an element consists of an opening and a closing tag. Some tags don't close but we'll get to that in a minute.

  • Opening tag: Tells the browser that you're about to start describing some sort of content. It could be a paragraph, or a link, or anything that a browser understands.
  • Content: The stuff you're describing! Usually this ends up being the words on the page.
  • Closing tag: When you're done with the content, you just match the opening tag with a closing tag.

Practice: <p>, <strong>, <em>.

Nesting tags

Elements can be nested. And, in practice, that's a critical part of HTML. We refer to the whole page, or the entire HTML file as the document. And, as you can see, it forms a tree. There are rules about what elements can go inside other elements but it's not strongly enforced and mostly obvious.

Practice: <html>, <head>, <body>, <ul>, <ol>, <li>.

Attributes

How would you go about linking to something? Even if you knew the element's tag was: <a>. Where would you put the link?

A complete list of all tags and their attributes can be found on the Mozilla Developer Network (MDN): HTML element reference.

Practice: <img>, <a>.

Some important keyboard shortcuts

When writing HTML there are two keyboard shortcuts you should use all the time:

  1. Command + Option + . will close the last opened tag
  2. Any tag name + Tab will stub out that tag for you

Try it. Note that for these to work, you must see "HTML" in the lower right of Atom.

A complete list of shortcuts is available here.
